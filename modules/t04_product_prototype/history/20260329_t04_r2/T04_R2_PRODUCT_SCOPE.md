# T04_R2_PRODUCT_SCOPE

## 1. 目标与边界

- 第二阶段目标是把第一阶段的原型闭环升级为“高可信产品原型”
- 本轮不推翻 `P2 / P3 + Flow A / B / C`
- 本轮只补：
  - `P4 越野地图页`
  - `P4-overlay 安全抽屉`
  - `P5 越野总结页`
  - Storybook 完整原型站

## 2. 页面升级范围

- `P4 越野地图页`：从占位版升级为正式原型骨架
- `P4-overlay 安全抽屉`：从通用占位块升级为可区分模式的正式原型覆盖态
- `P5 越野总结页`：从收口占位页升级为“证据转结论”的总结原型

## 3. 正式需求 vs 原型占位

### 3.1 P4 越野地图页

- 正式需求：
  - 三模式共用一套页面结构
  - 顶部轻状态栏、中央地图主画布、右侧 5 个固定入口、底部轻信息条、底部安全抽屉覆盖态
  - 普通态 / 3D 态切换
  - 四级警示体系可切换展示
- 原型占位：
  - GPS、记录、卫星通讯、偏离判断、停滞判断都只做 mock / derived 状态
  - 回到当前位置、寻迹返航、回接驳点、卫星通讯入口都只做原型动作，不做真实能力

### 3.2 P4-overlay 安全抽屉

- 正式需求：
  - 仍为地图页覆盖态，不拆独立页面
  - 固定 3 区块：
    - 当前风险状态区
    - 保守动作区
    - 外部联络与标记区
  - 平台 / 导入 与自由探索文案必须区分
- 原型占位：
  - 风险算法、返航路径、通讯链路、风险上报只做入口和说明

### 3.3 P5 越野总结页

- 正式需求：
  - 左侧 5 张固定卡片
  - 右侧地图回放区
  - 地图下方轻量时间轴 / 传感器摘要条
  - 左侧底部固定 3 个动作
  - 三模式文案差异成立
- 原型占位：
  - 传感器全部为 derived / mock，不伪装成真实车端
  - 视频、3D 回放、分享资产只保留入口
  - 不做真实回放引擎、真实视频生成、真实候选 Route 提交流程

## 4. 模式差异

- 平台 Route：强调平台参考线、接驳点 / 回撤点、平台 Route 一致性反馈
- 导入轨迹：强调导入轨迹 / 自定义参考路线、回撤点、保存为自定义路线 / 提交候选 Route
- 自由探索：强调探索范围、安全锚点、已走轨迹、私有探索 / 提炼候选 Route
