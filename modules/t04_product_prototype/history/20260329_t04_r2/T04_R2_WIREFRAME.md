# T04_R2_WIREFRAME

## 1. 线框基准

- 线框基于横屏平板原型壳输出，页面以“固定边框 + 内部可滚动区”的方式组织。
- 本文只冻结区块位置、尺寸关系、固定与滚动规则，不定义最终视觉样式。

## 2. P4 越野地图页线框

### 2.1 主线框

```text
+----------------------------------------------------------------------------------+
| 顶部轻状态栏                                                                     |
| [返回] [模式标签]                               [离线] [GPS] [记录] [卫星通讯]    |
+----------------------------------------------------------------------------------+
|                                                                                  |
|                                  地图主画布                                      |
|  - 卫星底图                                                                       |
|  - 当前定位 / 朝向                                                                |
|  - 已走轨迹                                                                       |
|  - 参考线或探索范围                                                               |
|  - 起点 / 终点 / 安全锚点 / 原型锚点                                              |
|                                                                      [当前位置]   |
|                                                                      [2D/3D]     |
|                                                                      [关键点]    |
|                                                                      [安全入口]  |
|                                                                      [结束 Trip] |
|                                                                                  |
|----------------------------------------------------------------------------------|
| 警示条：Level N / 标题 / 一句说明 / 建议动作                                      |
|----------------------------------------------------------------------------------|
| 底部轻信息条： [指标 1] [指标 2] [当前提示]                                      |
+----------------------------------------------------------------------------------+
```

### 2.2 固定与滚动

- 顶部轻状态栏固定，不滚动。
- 地图主画布固定为页面主区域，不出现页面级纵向滚动。
- 右侧浮动操作区固定在地图右侧，垂直排列。
- 警示条固定在底部轻信息条上方。
- 底部轻信息条固定在底部。
- 页面主滚动只允许发生在安全抽屉内部，不发生在地图页主体。

### 2.3 三种模式的线框差异

#### 平台 Route

```text
地图对象：参考线 + 已走轨迹 + 起点 + 终点 + 接驳点 + 回撤点 + 关键锚点 + 安全锚点
底部条：距离下一关键点 / 距离终点 / 沿参考路线继续前行
```

#### 导入轨迹

```text
地图对象：导入轨迹参考线 + 已走轨迹 + 起点 + 终点 + 回撤点 + 关键锚点 + 安全锚点
顶部标签：导入轨迹 / 自定义参考路线
底部条：距离下一关键点 / 距离终点 / 沿参考轨迹继续前行
```

#### 自由探索

```text
地图对象：探索范围 + 已走轨迹 + 起点 + 安全锚点 + 手动标点 + 当前定位
底部条：距离起点 / 距离最近安全锚点 / 处于范围内或接近边界
```

### 2.4 四级警示在线框中的落位

| 级别 | 线框位置 | 结构表达 |
| --- | --- | --- |
| Level 1 | 底部轻信息条内部 | 当前提示切换，不增额外层 |
| Level 2 | 警示条 | 黄条或高亮条，保留地图可视面积 |
| Level 3 | 警示条 + 右侧安全入口高亮 | 警示条更强，安全入口进入强调态 |
| Level 4 | 警示条 + 顶部卫星通讯状态强调 + 安全抽屉优先入口 | 页面仍停留在地图，不跳转新页 |

## 3. P4-overlay 安全抽屉线框

### 3.1 抽屉结构

```text
                    地图背景保留可见

        +------------------------------------------------------------+
        | 安全抽屉                                           [关闭]  |
        |------------------------------------------------------------|
        | 当前风险状态区 | 保守动作区                | 外部联络与标记区 |
        |--------------|---------------------------|------------------|
        | 状态标签      | 动作 1                    | 风险标记入口      |
        | 一句说明      | 动作 2                    | 发送当前位置      |
        |              | 动作 3                    | 卫星通讯入口      |
        +------------------------------------------------------------+
```

### 3.2 尺寸与滚动规则

- 抽屉默认高度建议为页面高度的 `42% ~ 48%`。
- 抽屉内可以纵向滚动，但第一屏必须完整露出：
  - 当前状态标签。
  - 第一个保守动作。
  - 卫星通讯入口。
- 抽屉不能完全遮住顶部状态栏。

### 3.3 模式差异

#### 平台 Route / 导入轨迹

```text
保守动作区顺序：
1. 寻迹返航
2. 回接驳点 / 回回撤点
3. 回终点
```

#### 自由探索

```text
保守动作区顺序：
1. 沿已走轨迹返回
2. 回起点
3. 回最近安全锚点
```

## 4. P5 越野总结页线框

### 4.1 主线框

```text
+--------------------------------------+-------------------------------------------+
| 左侧头部                             | 右侧头部                                  |
| [返回]                               | 地图回放区标题                             |
| 路线名称（最多 2 行）                | [2D/3D]                                   |
| [模式标签] [完成状态] [挑战指数]     | 当前高亮路段名称                           |
|--------------------------------------|-------------------------------------------|
| 左侧内容滚动区                        |                                           |
| 1. 总览卡                             |                                           |
| 2. 路线画像卡                         |               地图回放区                  |
| 3. 关键路段卡                         |   - 实际轨迹                               |
| 4. 车辆与挑战表现卡                   |   - 起点 / 终点 / 回撤点                   |
| 5. Route 反哺 / 分享卡                |   - 关键事件锚点                           |
|                                      |   - 当前高亮路段                           |
|--------------------------------------|-------------------------------------------|
| 左侧底部固定操作区                    | 时间轴 / 传感器摘要条                      |
| [保存总结] [生成分享内容] [反馈/候选] | [时间轴滑杆 + 事件点] [2~3 条摘要]         |
+--------------------------------------+-------------------------------------------+
```

### 4.2 固定与滚动

- 左侧头部固定。
- 左侧五张卡片区域为主滚动区。
- 左侧底部三按钮固定。
- 右侧头部固定。
- 右侧地图回放区固定占右侧主高度。
- 右侧底部时间轴 / 传感器摘要条固定在地图下方，不独立跳页。

### 4.3 左侧五张卡片的线框层级

#### 1. 总览卡

```text
[模式] [完成状态]
[总里程] [总时长]
[开始时间] [结束时间]
[回撤情况]
```

#### 2. 路线画像卡

```text
[风格标签] [风格标签] [挑战指数]
一句话路线画像总结
```

#### 3. 关键路段卡

```text
[最难一段]
[最精彩一段]
[最需谨慎一段]
点击任一项 -> 右侧地图高亮 + 时间轴跳到对应事件
```

#### 4. 车辆与挑战表现卡

```text
项目 1：结论值 + derived/mock 标记
项目 2：结论值 + derived/mock 标记
项目 3：结论值 + derived/mock 标记
项目 4：结论值 + derived/mock 标记
```

#### 5. Route 反哺 / 分享卡

```text
模式相关反馈结论
深度总结入口（占位）
分享卡 / 视频 / 3D 回放入口
```

### 4.4 右侧地图回放区与时间轴条

```text
右侧头部
- 当前高亮路段名称
- 2D / 3D 切换

地图区
- 实际轨迹常驻
- 关键路段高亮可切换
- 起点 / 终点 / 回撤点 / 关键事件锚点

底部时间轴 / 摘要条
- 一条轻量时间轴滑杆
- 若干关键事件打点
- 当前时间标签
- 右侧 2~3 条传感器结论摘要
```

### 4.5 三种模式在 P5 的线框差异

| 区块 | 平台 Route | 导入轨迹 | 自由探索 |
| --- | --- | --- | --- |
| 左侧第 5 卡 | 提交路线反馈 | 保存为自定义路线 / 提交候选 Route | 保存为私有探索 / 提炼候选 Route |
| 路线画像 | 强调与平台路线的一致性 | 强调自定义参考价值 | 强调探索风格与边界控制 |
| 地图高亮 | 对照平台参考完成情况 | 对照导入轨迹参考完成情况 | 回看实际探索过程 |

## 5. 联动规则

- `P4`：
  - 右侧 `关键点开关` 只影响关键点类对象显隐。
  - `安全入口` 打开 `P4-overlay`，关闭后回到同一地图状态。
- `P5`：
  - 点击左侧关键路段卡，右侧地图立即切换到对应高亮路段。
  - 点击时间轴事件点，地图高亮与传感器摘要同步更新。
  - 拖动时间轴滑杆，至少更新当前时间标签与右下摘要。

## 6. 主线程实现约束

- `P4` 不应再保留独立说明卡或大段页面介绍。
- `P4-overlay` 必须保持三栏逻辑，不扩展为多层菜单。
- `P5` 必须体现“左侧结论、右侧证据”的结构，不能退化成纯统计面板。
- `P5` 左侧底部必须是 3 个固定动作，不再混入 Demo Launcher 导航。
