# 04 方案策略

## 状态

- 当前状态：`模块级方案策略说明`
- 来源依据：
  - 当前任务书
  - `INTERFACE_CONTRACT.md`

## 主策略

1. 先做仓库规范发现和 T02 合同冻结
2. 先用真实设备验证单条候选的“列表 -> 详情 -> 轨迹证据”闭环
3. 只接受“双重筛选”后的路线进入正式批量清单：
   - 平台级显式越野信号
   - 详情语义确认驾车越野
4. 先保全原始证据，再归一化输出草案包
5. 用统一打包入口生成 Route 索引、QA 汇总和逐路线目录

## 筛选策略

### Gate A：显式标签

- `2bulu` 页面必须能回查到 `越野车` 标签
- `liuzhijiao` 页面必须能回查到 `越野` 搜索命中和详情页越野车辆信号

### Gate B：详情语义

- 详情文本、标题、风险说明、关键点或路线说明必须支持“驾车越野”判断
- 若只看到 `步道 / 徒步 / 登山` 主语义，直接排除

## 几何策略分层

### Primary

- 官方导出 `GPX / KML / KMZ`

### Secondary

- 详情页或轨迹页上可回查的结构化几何文件

### Fallback

- 只有轨迹截图时，允许 `screen_reconstructed`
- 但必须显式标注低一档几何置信度，并补风险说明

- 运行步骤写入 `RUNBOOK.md`
