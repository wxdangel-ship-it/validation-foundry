# 阶段 Gate 方案

## Gate 记录统一要求

每个阶段结束后都必须汇总并落盘以下内容：

1. 当前阶段目标
2. 实际完成情况
3. 是否通过 gate
4. 关键证据位置
5. 下一阶段是否允许继续
6. 遗留问题与风险
7. 若失败，失败原因分类与建议处理

## Phase 0

- 目标：项目初始化、规范继承、文档骨架建立
- 通过条件：
  - 项目级治理骨架已建立
  - T01 模块文档面已建立
  - `spec / plan / tasks` 已落盘
  - 子线程拆解和 gate 方案已写回仓库

## Phase 1

- 目标：安卓虚拟环境与虚拟定位单点打通
- 通过条件：
  - 明确模拟器方案、安装方式和版本
  - 明确 APK 获取与安装方案
  - 明确虚拟定位注入链路
  - 至少完成一个单点 smoke，并有证据

## Phase 2

- 目标：滴滴主链路打通或明确阻塞
- 通过条件：
  - 未登录前提下验证滴滴首页定位按钮链路
  - 明确是否可进入“附近上车点”状态
  - 对阻塞给出证据和失败分型

## Phase 3

- 目标：高德备选链路打通或明确阻塞
- 通过条件：
  - 验证进入打车服务后的定位按钮链路
  - 明确与滴滴差异
  - 明确是否可作为备选方案

## Phase 4

- 目标：上车点坐标提取规则定版与 GCJ-02 闭环验证
- 通过条件：
  - 目标点定义固定
  - 稳定判据可重复、可审计
  - 提取方法完成少量闭环验证
  - 输出坐标系与输入一致性的假设有证据支撑

## Phase 5

- 目标：浦东 `10` 点 golden set 跑通
- 通过条件：
  - `10` 个样例点已选定并记录
  - 每个样例都有结果记录、原因和证据目录
  - 用户可据此回查截图和操作证据

## Phase 6

- 目标：批量执行与结果导出
- 通过条件：
  - 从 CSV 读取批量输入
  - 输出统一结果 CSV
  - 每条记录有证据目录
  - 形成失败统计与摘要
