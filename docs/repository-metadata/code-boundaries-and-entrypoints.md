# 当前代码边界与执行入口约束

## 1. 文档目的

本文档用于说明当前仓库对“单文件体量”和“执行入口”的正式约束。

## 2. 约束范围

- 约束主体是纳入版本管理的源码与脚本文件，重点包括 `.py`、`.sh`、`.cmd`、`.ps1`、`.js`、`.ts` 等
- `tests/`、`tools/` 中同类源码与脚本也纳入体量审计

## 3. 单文件体量约束

- 单个源码或脚本文件超过 `100 KB` 时，视为结构债或超阈值文件
- 若必须继续修改超阈值文件，必须同时给出拆分计划或结构整改说明

## 4. 执行入口治理

### 4.1 什么算执行入口

当前把以下独立启动面视为执行入口：

- repo 级 `python -m validation_foundry`
- `scripts/`、`tools/` 下可直接执行的脚本
- 模块内带独立启动面的 `__main__.py`、`run.py` 或其它 CLI 文件

### 4.2 当前治理规则

- 默认禁止新增新的执行入口脚本
- 只有在现有入口无法通过参数化或模块复用解决时，才允许新增
- 任何新增执行入口都必须登记到 `entrypoint-registry.md`

## 5. 当前阶段说明

- Phase 0 仅建立包骨架，不冻结任何正式 CLI
- 首个正式入口应在 Phase 1 或 Phase 2 打通后再登记
