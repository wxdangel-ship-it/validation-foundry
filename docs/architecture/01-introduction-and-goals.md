# 01 引言与目标

## 状态

  - `docs/PROJECT_BRIEF.md`

## 系统目标

`validation-foundry` 的目标是建立一个可复现、可审计、可批量执行的虚拟手机验证工程底座，用于验证地图应用在虚拟定位后的上车点坐标表现。

## 业务目标

当前业务目标不是同时铺开多个模块，而是先让 `T01` 具备：

- 可初始化
- 可治理
- 可分阶段推进
- 可回查证据
- 可对单样本与小样本做可信闭环

## 文档目标

当前文档治理目标是：

- 明确项目级与模块级 source-of-truth 分层
- 把 `AGENTS.md` 约束在 durable guidance
- 把阶段计划、状态、gate 和子线程 writeback 固化为仓库文件
- 为未来新增模块保留统一模板，但当前不启用其他业务模块
